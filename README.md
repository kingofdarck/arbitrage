## 🎯 Особенности

- 🧠 **Умная дедупликация** - отправляет только НОВЫЕ возможности
- 📱 **Раздельные уведомления** - межбиржевой и треугольный арбитраж отдельно
- 📊 **Качественные сигналы** - минимальная прибыль 0.75%
- 🆓 **Бесплатный хостинг** - Railway (500 ч/мес), Render, Codespaces
- 🔄 **Автоматический перезапуск** при сбоях
- 🌐 **Health check API** для мониторинга

## 🚀 Быстрый старт (5 минут)

### 1. Создайте Telegram бота
- Напишите @BotFather → `/newbot`
- Скопируйте токен: `1234567890:ABCdefGHI...`
- Напишите боту сообщение
- Получите chat_id: `https://api.telegram.org/bot<ТОКЕН>/getUpdates`

### 2. Деплой на Railway (рекомендуется)
1. Нажмите [![Deploy on Railway](https://railway.app/button.svg)](https://railway.app/template/smart-arbitrage-monitor)
2. Войдите через GitHub
3. Добавьте переменные:
   - `TELEGRAM_BOT_TOKEN` = ваш токен
   - `TELEGRAM_CHAT_ID` = ваш chat_id
4. Нажмите "Deploy"

### 3. Готово! 🎉
Система запустится автоматически и начнет отправлять уведомления в Telegram.

## ⚙️ Настройки

### Изменение порога прибыли
В `config.py`:
```python
MONITORING_CONFIG = {
    'min_profit_threshold': 1.0,  # Повысить до 1.0%
}
```

### Изменение интервала проверки
В `smart_arbitrage_monitor.py`:
```python
await monitor.run(check_interval=30)  # Каждые 30 секунд
```

## 📈 Статистика

- **~5000 торговых пар** мониторится
- **Проверка каждые 10 секунд**
- **Фильтрация по белому списку** качественных криптовалют
- **Отслеживание истории** возможностей (2 часа)
- **Автоматическая очистка** устаревших данных

## ⚠️ Важные замечания

### Риски
- Цены могут измениться к моменту исполнения
- Комиссии за вывод могут съесть прибыль
- Ликвидность может быть недостаточной

### Рекомендации
- Начните с небольших сумм
- Проверяйте возможности вручную
- Учитывайте все комиссии бирж
- Следите за временем перевода средств

## 🆘 Поддержка

### Проблемы с Telegram:
- Проверьте правильность токена и chat_id
- Убедитесь, что бот не заблокирован

### Проблемы с хостингом:
- Проверьте логи на платформе
- Убедитесь в правильности переменных окружения
- Проверьте health check endpoint

### Система не находит возможности:
- Это нормально при спокойном рынке
- Порог 0.75% довольно высокий для качественных сигналов
